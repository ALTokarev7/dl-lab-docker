# Neural Style Transfer

Данный репозиторий подготовлен в рамках выполнения лабораторной работы по курсу «Глубокое обучение». За основу взят проект, реализующий классический алгоритм переноса стиля изображения с использованием предобученной нейросети VGG19.

*   **Ссылка на оригинальный проект:** [gordicaleksa/pytorch-neural-style-transfer](https://github.com/gordicaleksa/pytorch-neural-style-transfer)

---

## Краткое описание окружения проекта

Для обеспечения переносимости, воспроизводимости и стабильного запуска проекта на различных архитектурах окружение было переведено на легковесную конфигурацию без использования GPU:

*   **Базовый образ:** `python:3.10-slim` (минималистичный дистрибутив Linux Debian).
*   **Фреймворк:** `PyTorch` (сборка для CPU). Это позволило сократить размер итогового Docker-образа и обеспечить быстрый старт на любых устройствах.
*   **Обработка изображений:** Использован пакет `opencv-python-headless` вместо стандартной версии OpenCV. Это избавляет от необходимости установки тяжелых системных библиотек графического интерфейса (X11/GL) внутрь контейнера.
*   **Дополнительные зависимости:** `numpy`, `matplotlib`.
*   **Предобученные веса:** Веса сети `VGG19` скачиваются на этапе сборки Docker-образа (`docker build`), что позволяет запускать контейнер в офлайн-режиме непосредственно во время демонстрации работы.

## Инструкция по запуску проекта через Docker

Клонируйте репозиторий проекта: 
```bash
git clone git@github.com:ALTokarev7/dl-lab-docker.git
```

### 1. Сборка Docker-образа
Перед запуском необходимо собрать образ. Откройте терминал в корневой папке проекта и выполните команду:
```bash
docker build -t lab_nst .
```

### 2. Запуск теста на корректность и воспроизводимость
Для проверки работоспособности и воспроизводимости результатов запустите изолированный контейнер:
```bash
docker run --rm lab_nst
```
При запуске будет вызван тестовый скрипт на Python.
RunCheckerTest.py выполнит следующие шаги:
1. Зафиксирует random seeds (random, numpy, torch).
2. Запустит процесс переноса стиля на тестовых изображениях (lion.jpg и vg_starry_night.jpg) с ограничением в 300 итераций для быстроты прохождения теста. (разрешение также понижено)
3. Найдет сгенерированный результат в папке вывода.
4. Вычислит среднюю абсолютную ошибку (MAE) между полученным изображением и эталоном.
5. Выведет сообщение об успешном прохождении теста при соответствии установленным допускам.
